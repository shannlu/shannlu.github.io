A&M Chess Camp Registration Site

A fully-functioning chess camp registration site, featuring different use cases for roles such as parents, instructors, and guests. Allows users to create accounts, browse camps, register students for camp, and check out, among other features


LANGUANGE / SYSTEM
Ruby, Rails, HTML, CSS, JavaScript, SQLite

TERM
Spring 2018

The task -
To create a fully-functioning chess camp registration site from scratch - taking into account necessary functions for different roles, an intuitive user experience, and a live database.

The process involved understanding all application requirements, designing and building out the database, developing the application using an MVC model, and user testing to ensure that all functionality worked as expected.


Understanding application requirements & creating the database -
In order to begin, I first had to understand what the registration site would be used for and how it should function. At a high level, all users would be able to browse different chess camps offered, curriculums for the camps, locations the camps were held at, and instructors. Parents would be able to create an account and log in, add students to their family, select camps to register for, and check out. Instructors would be able to modify curriculum and course information, while guests would only be able to view public information. A detailed list of use cases for different types of users can be found here.

From the use cases, I had to determine what pieces of information were necessary to keep track of and how the pieces related to each other. Throughout the process of doing so, I created an entity-relationship diagram to represent the database structure.



Additional documentation: database design in third normal form & data dictionary

Having a comprehensive understanding of what information was stored in the database and how the information was organized played an integral part in building the rest of the application because it ensured that the correct information could be accessed efficiently.



Developing the application -
Using a Model-View-Controller (MVC) framework, I built the application using Ruby on Rails in phases that corresponded to those separate parts. A high-level overview of the steps I took to develop this web application is as follows:

  1. On the backend, the database entities were built out as models, with a full suite of unit tests to make sure that the data stored within each model was accurate and feasible (ie. phone numbers could only be 10 digits).

  2. Basic views were created to display information about the models - the pages that allow users to browse different camps, locations, instructors, and curriculums. Utilizing HTML, CSS, and Materialize, basic styling was included to appropriately visualize the information.

  3. Basic controller functionality was implemented, including the CRUD (create, read, update, delete) operations for all the entities. This allowed the users to update information about a model in the database.

  4. After functionality for viewing and editing camps, curriculums, locations, and instructors were completed, the rest of the models (users, registrations, students, camp instructors) were created.

  5. Following a similar process as before, the views and controllers for the new models were implemented. This included creating the process for more complex tasks - such as making an account, logging in, registering a student for a camp, making a payment / checking out, and assigning instructors to different camps.

  6. Once all user functions were feasible, I added authorization for different types of accounts. Admin accounts were able to view and edit information on all pages, instructor accounts could only change information for camps and curriculums they were assigned to, and parent accounts could only view information.

  7. User testing was conducted to make the end-to-end process as intuitive as possible. Changes to the structure and interactions were made based on user feedback.
